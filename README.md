# Avito Telegram Bot — Уведомления и ответы на сообщения Авито

## Описание проекта
Telegram-бот для владельца ломбарда (или любого бизнеса на Авито), который:
- Получает **уведомления о новых сообщениях** в мессенджере Авито **только по выбранным объявлениям** (item_id / "адресам").
- Позволяет **отвечать на сообщения прямо из Telegram** (через официальное Avito Messenger API).
- **Строгий контроль доступа**:
  - Бот доступен только владельцу по умолчанию.
  - Другие пользователи отправляют заявку → владелец получает inline-кнопки "Разрешить / Отклонить".
  - В админ-меню можно **ограничивать доступ** (например, уволенным сотрудникам — статус `restricted`).
- Всё работает **только через официальное Avito API** (никакого парсинга/скрейпинга).

## Подготовка (обязательно сделай перед запуском)

1. **Telegram Bot**
   - Напиши @BotFather → `/newbot` → сохрани токен.

2. **Avito Business API** (пока не обязательно для v0.1)
   - У тебя должен быть бизнес-аккаунт Авито с подпиской **Расширенный** или **Максимальный** (Messenger API доступен не на всех тарифах).
   - Перейди в ЛК Авито → Настройки → Интеграции и API.
   - Создай/скопируй **Client ID** и **Client Secret**.
   - Узнай свой `user_id` (можно будет через `/core/v1/accounts/self` после получения токена).
   - Разреши доступ к Messenger API.

3. **Окружение**
   ```bash
   python -m venv venv
   source venv/bin/activate  # или venv\Scripts\activate на Windows
   pip install -r requirements.txt
   cp .env.example .env
   # отредактируй .env
   ```

## Как запустить (v0.2)

1. Распакуй архив.
2. Перейди в папку проекта:
   ```bash
   cd avito_telegram_bot
   ```
3. Создай виртуальное окружение и установи зависимости:
   ```bash
   python -m venv venv
   source venv/bin/activate          # Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```
4. Скопируй настройки:
   ```bash
   cp .env.example .env
   ```
5. Открой `.env` и обязательно заполни:
   - `BOT_TOKEN` — токен от @BotFather
   - `OWNER_TELEGRAM_ID` — твой Telegram ID (узнай у @userinfobot)

6. Запусти бота:
   ```bash
   python main.py
   ```

Готово! Открой бота в Telegram → `/start`.

## Как будет работать в полной версии (план)

### Архитектура
- **main.py** — точка входа, запуск бота + фоновой задачи polling.
- **config.py** — токены, OWNER_TELEGRAM_ID, пути к БД.
- **database.py** — функции работы с SQLite (users, monitored_ads, chat_states).
- **avito_client.py** — класс для работы с Avito API:
  - Получение/обновление access_token (client_credentials).
  - `get_chats()`, `get_messages(chat_id)`, `send_message(chat_id, text)`, `get_item_info(item_id)`.
- **handlers/** — обработчики сообщений и callback'ов.
  - `start.py`, `admin.py`, `avito_handlers.py` (уведомления, FSM для ответа).
- **services/notifier.py** — фоновая задача (каждые 20-60 сек):
  - Опрашивает Avito → новые/непрочитанные сообщения по `monitored_ads` (только enabled).
  - Отправляет уведомления **всем approved пользователям** (или по настройкам).
  - Кнопки в уведомлении: "Ответить", "Прочитать чат", "Пропустить".
- **FSM** для ввода текста ответа.

### "Адреса" / Объявления
- В боте владелец (и админы) может:
  - Добавить объявление по ссылке Авито или item_id.
  - Бот сам получит название объявления через Items API.
  - Включать/выключать уведомления по каждому объявлению (inline кнопки).
  - Список активных объявлений.

### Уведомления и ответы
- Уведомление примерно так:
  ```
  🆕 Новое сообщение по объявлению:
  "iPhone 13 128GB" (item_id: 123456789)

  От: Иван П. (покупатель)
  Текст: Здравствуйте, можно посмотреть сегодня?

  [Ответить] [Прочитать] [Детали чата]
  ```
- При нажатии "Ответить" → бот запрашивает текст → отправляет в Авито API → подтверждает.

### Ограничения доступа
- Статусы пользователей: `pending` → `approved` / `restricted`
- Владелец (OWNER_TELEGRAM_ID) всегда имеет полный доступ + админку.
- restricted пользователи видят "Доступ ограничен. Обратитесь к владельцу."
- Можно расширить: роли (admin/user), кто может добавлять объявления и т.д.
